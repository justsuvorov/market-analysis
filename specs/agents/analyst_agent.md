# Спецификация: AnalystAgent

## Status
`Draft`

## Purpose
Агент-аналитик. Принимает сырые данные от `ResearcherAgent`,
проводит структурированный анализ и формирует финальный отчёт.

## Input
**Модель:** `RawResearchData` (выход `ResearcherAgent`)

## Output
**Модель:** `ResearchReport` (см. `specs/marketing_research_v1.md`)

## Задачи агента

### 1. Конкурентный анализ
- Структурировать 10–15 объектов в `CompetitorProfile`
- Выделить ≥2 неудачных кейса, разобрать ошибки
- Параметры: локация, формат, цены, инфраструктура, рейтинги, загруженность

### 2. Скоринг локаций (CatBoost + GLM)
- Оценить 3–4 локации по взвешенной матрице критериев
- GLM: вывести коэффициенты влияния каждого фактора (для бизнес-отчёта)
- Результат: `LocationScore` с баллами и рекомендацией

### 3. Сегментация ЦА
- Три сегмента: инвесторы в строительство, покупатели домов, арендаторы
- Для каждого: демография, мотивация, критерии выбора, барьеры
- Результат: `AudienceSegment` с детальным портретом

### 4. Варианты концепции
- 2–3 варианта с УТП, форматом позиционирования, архитектурными ориентирами
- Сравнительная оценка перспективности

### 5. Вопросы для экспертных интервью
- Гайд для 4 экспертов (см. `specs/marketing_research_v1.md`)
- Темы: CAPEX/OPEX, кассовый разрыв, риски, привлечение инвестиций

### 6. SWOT-анализ
- Учитывать рыночные факторы, локацию, выбранную концепцию

### 7. Финальный отчёт
- Аналитическая записка: 15–20 стр., Markdown → PDF
- Презентация: 12–15 слайдов

## ML подход
- **CatBoost Regressor:** скоринг локаций (target = инвестиционная привлекательность)
- **CatBoost Classifier:** сегментация конкурентов по формату/ценовому сегменту
- **GLM (statsmodels OLS):** интерпретация весов признаков для бизнес-аудитории
- Все модели сохранять в `src/ml/`, конфиги — в `src/ml/configs/`

## Constraints
- Все промежуточные результаты передавать через типизированные Pydantic-модели
- Каждый этап логировать и сохранять в `data/processed/`
- Финальный отчёт — только после явного подтверждения пользователя

## Решения
- Формат выходного отчёта: Markdown
- Карта локаций: не нужна (текстовая матрица достаточно)
